# Legacy Specification Archive

This directory contains specifications from the project's previous development workflow (SDD / Software Design Document).

**This is a read-only archive.** The active specification source is now:

- **`openspec/specs/`** — canonical baseline specifications
- **`openspec/changes/`** — active change proposals and implementations

## Historical Reference Only

Files in this directory (`api/`, `db/`, `product/`, `rfc/`, `testing/`) are preserved for:

- Understanding design decisions made during earlier development phases
- Tracing the evolution of the project's architecture
- Historical reference when investigating why specific patterns were chosen

They are **not** the source of truth for:

- How to implement new features
- What the current API contract is
- How to run the build or tests

For new development and changes, consult `openspec/specs/` and the active implementation in `include/`, `src/`, `tests/`, and `docs/`.
