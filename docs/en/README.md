# Documentation Hub

TensorCraft-HPC is documented as a **reference repository**: enough detail to build, understand,
and validate the project without turning the docs into a second drifting codebase.

## Start here

| Goal | Recommended path |
|------|------------------|
| Understand the project | [`README.md`](../../README.md) |
| Build and package locally | [`getting-started/installation.md`](getting-started/installation.md) |
| Understand structure | [`guides/architecture.md`](guides/architecture.md) |
| Explore API surface | [`api/README.md`](api/README.md) |
| Contribute safely | [`reference/contributing.md`](reference/contributing.md) |

## Documentation map

- `getting-started/` — installation, troubleshooting, local validation
- `guides/` — architecture and optimization notes
- `api/` — module-oriented API surface
- `examples/` — representative usage examples
- `reference/` — thin wrappers around canonical governance documents

## Workflow note

OpenSpec is the active workflow for repository changes:

1. accepted baseline: `openspec/specs/`
2. in-flight work: `openspec/changes/`
3. historical archive: `specs/`
