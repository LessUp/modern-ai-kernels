# 文档中心

TensorCraft-HPC 的文档按**参考型仓库**来组织：信息足够构建、理解、验证项目，但不过度堆积成
另一套会持续漂移的“平行代码库”。

## 从这里开始

| 目标 | 推荐入口 |
|------|----------|
| 了解项目定位 | [`README.zh-CN.md`](../../README.zh-CN.md) |
| 本地构建与打包 | [`getting-started/installation.md`](getting-started/installation.md) |
| 理解结构 | [`guides/architecture.md`](guides/architecture.md) |
| 查看 API 表面 | [`api/README.md`](api/README.md) |
| 安全参与贡献 | [`reference/contributing.md`](reference/contributing.md) |

## 文档结构

- `getting-started/` — 安装、故障排除、本地验证
- `guides/` — 架构与优化说明
- `api/` — 按模块划分的 API 文档
- `examples/` — 代表性使用示例
- `reference/` — 指向仓库根规范文档的轻量包装页

## Workflow 说明

OpenSpec 是当前主动开发流程：

1. 已接受基线：`openspec/specs/`
2. 正在进行的工作：`openspec/changes/`
3. 历史归档：`specs/`
